# Expected Outcome

The only allowed changes are:

1. `case/workspace/branding.md`: change "Open Code" -> "OpenCode" (one-line content change).
2. `case/workspace/version.txt`: change `0.0.1` -> `0.0.2`.

Process expectations (for scoring):

- A checkpoint commit occurs before edits.
- The model reads the files before editing.
- The model patches rather than rewrites.
- The model reviews a diff after editing.
- A final commit captures the changes.
