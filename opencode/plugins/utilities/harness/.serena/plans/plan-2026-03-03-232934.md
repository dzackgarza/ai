# Plan: Tester-First `opx` Harness (Runs + Transcripts)

## Goals
- Provide a tester-first harness CLI for OpenCode plugin developers.
- Support two primary testing workflows:
  1) One-shot runs: prompt -> agent actions -> idle -> transcript -> cleanup.
  2) One-shot runs with background continuation: prompt -> idle -> further events/actions -> idle -> transcript -> cleanup.
- Support resuming an existing session for multi-turn testing.
- Provide session introspection (idle/working, current tool, last tool, last error) using internal session/message/event data; do not parse transcripts for status.
- Make provider unavailability (rate limit/quota/auth) explicit so tests do not churn on false failures.

## Non-Goals
- Do not make testers reason from machine-friendly JSON message graphs as the primary artifact.
- Do not expose implementation plumbing flags (`--async`, `--no-reply`) in the public surface.
- Do not attempt to infer status by scraping human transcript text.

## Canonical Tester Artifact: Transcript
- Testers validate behavior based on the human-visible transcript.
- Transcript generation uses the existing transcript tooling that takes a session ID as input and produces a text file.
- `opx` should:
  - Generate transcript by invoking the transcript parser script with the session ID.
  - Save transcript to a text file (default: `/tmp/transcript-<session_id>.txt`).
  - Print the transcript contents to stdout and print the saved path.

### Transcript Script Configuration
- Add support for:
  - `OPX_TRANSCRIPT_SCRIPT` to override the script path.
- Default script path (current repo behavior):
  - `/home/dzack/.config/agents/skills/reading-transcripts/scripts/parse_opencode_log.py`
- Failure mode:
  - If transcript generation fails, keep the session (do not delete), print session ID, and exit non-zero.

## Public CLI Surface

### 1) One-shot Run (default deletes session)
`opx run --prompt "..." [--agent A] [--model provider/model] [--timeout SEC] [--linger SEC] [--keep]`

Behavior:
1. Create a fresh session.
2. Send the prompt.
3. Wait until the session reaches idle (internal signal, see Waiting Logic).
4. If `--linger` is set: keep listening for session activity and only exit when:
   - the session is idle, and
   - there has been no new activity for `linger` seconds,
   bounded by `--timeout`.
5. Generate transcript text file via transcript script (input: session ID).
6. Print transcript to stdout and print `Transcript saved to: <path>`.
7. Delete session unless `--keep`.

Notes:
- One-shots should generally delete sessions immediately after transcript generation.
- `--keep` is required to preserve a session for multi-turn testing.

### 2) Resume Existing Session (default deletes session)
`opx resume --session ses_... --prompt "..." [--timeout SEC] [--linger SEC] [--keep]`

Behavior:
1. Introspect current session status (working/idle, current tool, last tool, last error).
2. If session is busy/working, wait for idle (bounded by `--timeout`).
3. Send the new prompt.
4. Wait for idle / linger behavior as in `run`.
5. Generate transcript file via transcript script (input: session ID).
6. Print transcript to stdout and print saved path.
7. Delete session unless `--keep`.

### 3) Session Management / Introspection
- `opx session list`
- `opx session status --session ses_... [--watch]`
  - Shows:
    - state: `working|idle`
    - current running tool (if any)
    - last tool (completed/error)
    - last error classification (rate_limit/quota/auth/aborted/other)
  - Computed from internal events + message parts; no transcript parsing.
- `opx session transcript --session ses_... [--lines N|--full]`
  - Produces transcript using the transcript script by session ID.
  - `--lines N` may tail the produced file for convenience.
- `opx session abort --session ses_...`
- `opx session delete --session ses_...`

### 4) Provider Availability / Rate-Limit Signaling
- `opx provider check --provider <key>`
  - Strict known-pattern checks for providers/models known to rate-limit/quota.
  - Output is human-readable; exit code distinguishes provider-unavailable.

## Debug CLI Surface
- Move probes/trace utilities under `opx debug ...`.
- Keep any low-level flags like `--async`/`--no-reply` only in debug (if retained at all).

## Waiting Logic (Deterministic Idle + Linger)
- Prefer event-driven waiting via `client.event.subscribe`.
- Primary idle signal: `session.idle` event for the session.
- Activity signals (used to reset linger quiet timer):
  - `message.part.updated`
  - `message.updated`
  - `session.updated`
  - `session.error`
  - tool parts observed as running/completed/error
- Fallback when SSE fails: poll `session.messages` and infer tool running from message parts; still treat transcript generation as canonical output.
- Always bounded by `--timeout`.

## Session Status Computation (No Transcript Parsing)
- Derive status from internal data:
  - If any latest tool part has `state.status="running"`, session is `working` and report that tool.
  - If a `session.idle` event was most recently observed and no tool is running, session is `idle`.
  - Track last completed/error tool part as `last tool`.
  - Track `assistant.error` and `session.error` as `last error`.

## Provider Unavailability and Exit Codes
- Classify unavailability as: `rate_limit`, `quota`, `auth` (plus `aborted`, `other`).
- Exit code policy:
  - `0`: success
  - `1`: failure (non-provider)
  - `2`: provider unavailable (rate limit/quota/auth)
- `run`/`resume` should still print transcript; append a short footer describing provider unavailability when detected.

## Deletion Semantics
- Default: delete session after transcript is successfully generated and printed.
- If transcript generation fails: keep session and print session ID.
- `--keep`: never delete the session.
