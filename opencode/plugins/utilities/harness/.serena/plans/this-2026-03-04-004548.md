# Plan: Tester-First `opx` Harness (Run/Resume + Transcript Artifacts) v2

This plan is derived from the requirements clarified in chat: testers validate against the human-visible transcript text; sessions are ephemeral by default; keeping sessions is explicit.

## 1) Requirements (from chat)

### 1.1 Primary workflows
1) **One-shot (default)**
- `prompt -> agent actions -> idle -> transcript -> delete session`
- Must delete sessions **by default**.
- Must produce the transcript **before** deletion.

2) **Interactive hack (one-shot + background continuation)**
- `prompt -> agent actions -> idle -> further events -> further agent actions -> idle -> transcript -> delete session`
- Needs:
  - overall timeout for the entire chain
  - a post-idle "linger" window so we do not exit immediately at the first idle

3) **Resume existing session (multi-turn)**
- Same as one-shot / linger, but starting from an existing session.
- If the session is busy, provide tools to understand what it is doing and to wait until it is safe to send.

### 1.2 Tester-visible artifacts
- Testers should not reason based on machine-friendly output.
- Test output is the **transcript text** (what a real user would see).
- Save transcript to a text file and print the path so an agent/tester can read it.

### 1.3 Introspection (internal, not transcript-derived)
- Determine:
  - idle vs working
  - current tool (if running)
  - last tool (completed/error)
  - last error and an error-kind classification
- Must use internal session/message/event state. Do not parse transcripts for status.

### 1.4 Provider-unavailable signaling
- Tests must explicitly indicate when failures are due to provider rate-limit/quota/auth.
- Must use a distinct exit code for provider-unavailable to avoid incorrect "fixes" and churn.

## 2) Public CLI surface

### 2.1 `opx run` (create a session automatically)
`opx run --prompt <text> [--agent <agent>] [--model <provider/model>] [--timeout <sec>] [--linger <sec>] [--keep] [--transcript-out <path>] [--no-print]`

Defaults
- `--timeout`: 180s
- `--linger`: 0s
- `--keep`: false
- transcript path default: `/tmp/transcript-<session_id>.txt`
- print transcript body to stdout by default (unless `--no-print`)

Behavior
1. Create new session.
2. Send prompt.
3. Wait for idle (and linger if requested).
4. Generate transcript file (see Section 4).
5. Print transcript to stdout (unless `--no-print`).
6. Print `Transcript saved to: <path>`.
7. Delete session unless `--keep`.

Exit codes
- `0`: run completed successfully
- `1`: run failed (timeout/aborted/unknown)
- `2`: provider unavailable (rate_limit/quota/auth)

Important lifecycle rule
- If transcript generation succeeded, the session is deleted by default **even on timeout/provider-unavailable**, unless `--keep`.
- If transcript generation fails, keep the session (print session ID for cleanup/debug).

### 2.2 `opx resume` (prompt an existing session)
`opx resume --session <ses_id> --prompt <text> [--agent <agent>] [--model <provider/model>] [--timeout <sec>] [--linger <sec>] [--keep] [--transcript-out <path>] [--no-print]`

Behavior
1. Compute status immediately (idle/working + current tool + last error).
2. If working, wait for idle (bounded by `--timeout`).
3. Send new prompt.
4. Wait for idle/linger.
5. Generate transcript; print transcript + saved path.
6. Delete session unless `--keep`.

### 2.3 `opx session ...` (inspection + cleanup)
- `opx session list`
- `opx session status --session <ses_id> [--watch] [--interval <sec>]`
  - human-readable status lines
- `opx session transcript --session <ses_id> [--lines <n>] [--out <path>]`
  - generates transcript via transcript script by session ID
  - `--lines` tails the produced text file for convenience
- `opx session abort --session <ses_id>`
- `opx session delete --session <ses_id>`

### 2.4 `opx provider check ...` (explicit rate-limit/quota detection)
- `opx provider check --provider <key> [--timeout <sec>]`
  - strict known-pattern checks using `config/known_limit_patterns.json`
  - exit code `2` if unavailable

- Everything probe/trace-oriented lives under `opx debug ...`.
- No public `--async` / `--no-reply` knobs.

## 4) Transcript generation (canonical tester output)

### Mechanism
- Use the existing transcript parser script that takes **session ID** as input:
  - `python <script> <session_id> > <outPath>`
- This is the same script used by `opencode/plugins/read-transcript.ts`.

### Configuration
- Env:
  - `OPX_TRANSCRIPT_SCRIPT` overrides script path
- Default script path:
  - `/home/dzack/.config/agents/skills/reading-transcripts/scripts/parse_opencode_log.py`

### Output
- default output path: `/tmp/transcript-<session_id>.txt`
- `--transcript-out` supports `{session_id}` templating
- Always print the saved path.

### Failure semantics
- If transcript generation fails:
  - keep session
  - print session ID
  - exit non-zero

## 5) Waiting + linger logic

### Preferred: SSE
- Use `client.event.subscribe` and filter to the session.
- Use internal session/message/tool state, not transcript parsing.

### Idle determination (internal)
- "Idle now" means:
  - we have observed an idle signal for the session after the prompt-send time, and
  - no tool part is currently in `running` state.

### Linger semantics (first-idle hold; no quiet-window resets)
- `--linger` is the maximum time to wait in the FIRST idle state before exiting.
- Algorithm for `run`/`resume` after sending the prompt:
  1) Wait for the session to reach idle (idle-1).
  2) If `linger == 0`: stop waiting immediately.
  3) If `linger > 0`: wait up to `linger` seconds for the session to LEAVE idle-1 (i.e., a new agent turn begins / tools start running).
     - If the session stays idle-1 for the full linger duration: stop waiting.
     - If the session leaves idle-1 within the linger window: ignore any remaining linger time, then wait until the session becomes idle again (idle-2), and stop waiting immediately at idle-2 (no second linger).

### Fallback: polling
- If SSE fails, poll `session.messages` and infer:
  - tool-running transitions (leaving idle-1) by scanning tool parts for `running`
  - idle by the same "idle now" rule above (idle signal if available, otherwise stability + no tool running)

## 6) Status computation (do not parse transcript)

Sources
- `client.session.messages({id})`
- optional short SSE window for freshest idle/error

Outputs (human-readable)
- `state`: working|idle
- `current tool`: latest tool part with status=running (tool name + start time)
- `last tool`: latest tool part completed/error (tool name + end time + status)
- `last error`: assistant.error or session.error, plus error-kind classification

Error-kind classification
- rate_limit | quota | auth | aborted | other

## 7) Provider-unavailable signaling

Within `run/resume`
- If provider-unavailable detected during the run:
  - print a short footer line after the transcript path
  - exit code `2`

Explicit preflight
- Document: CI/tests should optionally run `opx provider check --provider <key>` before expensive runs.

## 8) Session lifecycle (hard rule)
- Default: delete sessions after transcript generation succeeds.
- Keep sessions only with `--keep` (explicit).
- Keep session if transcript generation fails (so it can be debugged/cleaned up).
